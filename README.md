# Fake Review Detector

An AI-powered system that detects fake and AI-generated product reviews using fine-tuned transformers and explainable AI.

## 🚀 Live Demo
👉 **[Try it live](https://sentinel-7a2svvehzxusbvmemtpun7.streamlit.app/)**

---

## What it does

Paste any product review and instantly find out if it was written by a real person or generated by an AI. The system explains exactly why it made its decision using SHAP explainability.

---

## Results

### MR2 Amazon Dataset (40,431 reviews)

| Model | Accuracy | F1 Score | ROC-AUC | Cohen Kappa | MCC |
|---|---|---|---|---|---|
| Baseline TF-IDF + LR | 92.86% | 0.9281 | 0.9814 | 0.8572 | 0.8572 |
| Linguistic Features + GBM | 85.07% | 0.8522 | 0.9336 | 0.7015 | 0.7016 |
| Hybrid (TF-IDF + Linguistic) | 90.49% | 0.9038 | 0.9699 | 0.8098 | 0.8100 |
| **DistilBERT (fine-tuned)** | **97.82%** | **0.9782** | **0.9980** | **0.9561** | **0.9562** |

### OpSpam Benchmark (1,600 hotel reviews)

| Model | Accuracy | F1 Score |
|---|---|---|
| Ott et al. 2011 (SVM) — Published | 89.6% | 0.895 |
| Li et al. 2014 (Neural) — Published | 88.6% | 0.884 |
| **Our TF-IDF + LR** | **90.31%** | **0.9040** |
| Our DistilBERT | 85.31% | 0.8630 |

Our TF-IDF baseline outperforms two published papers on the OpSpam benchmark without any domain-specific tuning.

## How it works

```
Input Review
     │
     ├──► TF-IDF Vectorizer ──► Logistic Regression ──► P(fake)
     │
     ├──► Linguistic Features ──► Gradient Boosting ──► P(fake)
     │    burstiness · perplexity · vagueness · caps ratio · type-token ratio
     │
     └──► TF-IDF + Linguistic ──► Hybrid Model ──► P(fake)
                                        │
                                  DistilBERT (50% weight)
                                        │
                                  Adaptive Ensemble
                                        │
                                  SHAP Explanation
                                        │
                                  Final Verdict
```

---

## Key findings

- **type_token_ratio** is the strongest linguistic predictor — LLM fakes repeat words more
- **avg_sent_len** and **char_count** differ significantly between real and fake reviews
- Real reviews contain more exclamation marks than subtle LLM fakes — counter-intuitive finding
- Subtle LLM fakes (MR2) and spam fakes show distinct linguistic signatures requiring dual-distribution training
- TF-IDF outperforms DistilBERT on OpSpam — vocabulary patterns are more discriminative for short hotel reviews

---

## Dataset

- **MR2 Fake Reviews Dataset** — 40,431 Amazon reviews (20,216 real, 20,215 LLM-generated) across 10 categories
- **OpSpam** — 1,600 hotel reviews (800 real, 800 deceptive), standard benchmark
- **10 product categories**: Books, Electronics, Clothing, Toys, Sports, Home & Kitchen, Pet Supplies, Movies, Tools, Kindle

## Run locally

```bash
# Clone
git clone https://github.com/d4nushka/SENTINEL.git
cd SENTINEL

# Install
python -m venv venv
venv\Scripts\activate        # Windows
pip install -r requirements.txt

# Download dataset (Kaggle account required)
kaggle datasets download -d mexwell/fake-reviews-dataset -p data/ --unzip

# Run pipeline
python src/data_collector.py
python src/features.py
python src/train.py
python src/explain.py

# Launch app
streamlit run app.py
```
